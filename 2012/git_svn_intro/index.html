什么是Git

  • 分布式版本控制／软件配置管理软件
  • 林纳斯·托瓦兹为管理Linux内核开发所设计开发
  • 速度快、适合多分支开发模式

传统版本控制的问题

  • 团队可能有严格的提交规则，导致大家不敢小步提交，导致复杂功能几乎没有做版本控制；
  • 没有合适的share未完成代码的方法，或者：
  • 为了share代码在仓库里留下许多未完成的版本；
  • 无法离线工作

分布式版本控制的改变

  • 去中心化
  • 可离线工作
  • 强大的分支功能

去中心化

  • 避免单点fail
  • 适合分布式的协作
  • 对于某些项目来说是政治正确的
  • 特别适合开源模式（不同分支可以竞争）和基于开源的商业模式（上下游独立性）

可离线工作

  • 随时工作（在家里、在飞机上、火车上……）
  • 随时提交
  • 随时看日志

分支

  • 本地随便开分支
  • 在分支上切换的代价非常低
  • 分支合并比较容易
  • 鼓励尝试
  • 鼓励创新
